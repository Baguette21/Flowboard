I. Introduction

FlowBoard is a personal Kanban-style task management web application designed to help individuals and teams organize, prioritize, and track their work through an intuitive visual board system. Managing tasks across multiple projects often becomes difficult when relying on fragmented tools such as spreadsheets, messaging apps, or basic to-do lists. These tools typically lack real-time collaboration, visual workflow representation, and the flexibility to reflect individual work styles. FlowBoard was developed to address these pain points by providing a clean, structured, and responsive environment where work can be visualized, filtered, and managed with minimal friction.

The application is built around the core Kanban concept of boards, columns, and cards. Users can create multiple boards — each representing a distinct project or area of responsibility — and organize tasks as cards within customizable columns such as "To Do," "In Progress," "Review," and "Done." Cards can be enriched with descriptions, priority levels (Urgent, High, Medium, Low), due dates, color-coded labels, and assignees. Drag-and-drop reordering is supported for both cards and columns using a fractional indexing system, ensuring stable and precise ordering without full re-indexing.

FlowBoard uses Convex as its real-time backend, which enables live synchronization of all board data across multiple users and browser sessions. Authentication is handled through a secure email-and-password flow with email OTP (one-time password) verification via SMTP. Board owners can invite collaborators by email, manage their membership status, and control assignment permissions. When a task is assigned to another user, a notification is automatically generated and delivered within the application. An activity log records every action taken on a board, providing a transparent history of task creation, movement, updates, and deletions. FlowBoard also includes a search and filter panel for quick access to specific tasks, along with keyboard shortcuts and a toggleable light/dark theme for a comfortable and efficient user experience.

II. Problem Statement & Objectives

Individuals and teams frequently struggle to manage tasks effectively when their workflow is spread across multiple disconnected tools — from handwritten notes and chat messages to generic spreadsheets. Without a centralized system, tasks are easily forgotten, priorities become unclear, and deadlines are missed. Manual approaches also make it difficult to visualize the overall state of a project or to understand where bottlenecks occur. When teams collaborate, the absence of real-time updates forces members to rely on manual status checks and constant communication to stay aligned, which is both time-consuming and error-prone.

Furthermore, existing generic task tools often lack features tailored for Kanban-style workflows — such as drag-and-drop column and card ordering, per-board label systems, priority-based filtering, and built-in assignment notifications. Users either resort to overly complex enterprise tools with steep learning curves or settle for bare-bones applications that fail to support productive, focused work. FlowBoard was built to fill this gap by providing a focused, fast, and feature-rich Kanban board application that is simple enough for personal use yet powerful enough for small team collaboration.

To improve task organization and team productivity, the following objectives shall be achieved upon completion of the system:

  • Create a secure web-based Kanban board application with user registration and login using email and password, verified through OTP.
  • Allow users to create and manage multiple boards, each with a custom name and accent color, and mark boards as favorites.
  • Allow users to create, rename, recolor, reorder, and delete columns within their boards.
  • Allow users to create task cards with titles, descriptions, priority levels, due dates, color-coded labels, and assignees.
  • Implement drag-and-drop reordering of both cards and columns using fractional indexing for precise and stable ordering.
  • Provide real-time synchronization of all board data across users and browser sessions through Convex.
  • Allow board owners to invite collaborators by email, and manage their membership status and assignment permissions.
  • Implement a notification system that alerts users when a task is assigned to them.
  • Maintain a per-board activity log that records all significant actions such as card creation, movement, updates, and deletions.
  • Include a search and filter panel allowing users to search cards by text and filter by priority, label, and due date.
  • Support keyboard shortcuts for efficient navigation and quick actions within the application.
  • Provide a toggleable light and dark theme that persists across sessions using local storage.

IV. Scope of the Project

FlowBoard is a web-based Kanban task management application that provides users with a visual, board-centered interface for organizing and tracking their work. The system allows authenticated users to create and manage multiple boards, each representing a distinct project or workflow. Within each board, users can define columns that represent the stages of their workflow and populate them with task cards that move across stages as work progresses. Users can access the application from any modern web browser on desktop or mobile devices, and all board data is synchronized in real time across sessions using Convex.

The operations of the application are centered around two user roles: board owners and board members. Board owners have full control over their boards, including the ability to create, edit, and delete columns and cards; manage labels; invite and remove members; and delete the board entirely. Board members can view and interact with boards they have been invited to, and may be granted permission to assign tasks to other members by the board owner. Each user has an individual account secured through email-and-password authentication with OTP email verification, ensuring that only authorized users can access private board data.

For the features of the application, users can create task cards with rich details including titles, descriptions, priority levels, labels, due dates, and assignees. Cards and columns can be reordered via drag-and-drop using a fractional indexing system. The search and filter panel allows users to quickly locate cards by text content, priority flag, label, or due date range. An activity log records all significant board and card actions for full transparency. Notifications are generated when tasks are assigned to users, keeping team members informed without manual follow-up. The application also supports keyboard shortcuts for power users and a toggleable light/dark theme that is persisted locally. The scope does not currently include time tracking, file attachments, recurring tasks, or external calendar integrations.

VI. Database Design (Schema)

For the database design, FlowBoard uses Convex as its backend platform. Convex is a reactive Backend-as-a-Service (BaaS) built on top of a document database, providing real-time queries, type-safe mutations, and serverless functions. The schema is defined using Convex's defineSchema and defineTable utilities and consists of nine tables that collectively manage user accounts, board structures, task data, collaboration features, and system events.

The users table stores registered user accounts. Each record contains an optional authentication ID (authId), display name, profile image, email address, email verification timestamp, and an anonymity flag. Indexes are defined on the email, phone, and authId fields to support fast lookups during login and profile retrieval.

The boards table is the top-level organizational entity of the application. Each board record stores the owning user's ID, the board name, a URL-friendly slug, an accent color, a favorite flag, and creation and update timestamps. Boards are indexed by their owner's user ID and by slug to support fast board listing and slug-based navigation.

The boardMembers table manages the collaborative membership of boards. Each record links a board and a user, storing the ID of the user who sent the invitation, the timestamp of when the user joined, and an optional flag indicating whether the member has permission to assign tasks. This table is indexed by boardId and userId to support efficient membership lookups.

The boardInvites table tracks pending and responded invitations. Each invite record stores the board and invited email address, the ID of the inviting user, the current status of the invitation (pending, accepted, or declined), and both creation and response timestamps. Multiple indexes allow the system to efficiently query invitations by board, email, and status.

The columns table represents the workflow stages within a board. Each column record stores the parent board ID, a title, a fractional order key used for drag-and-drop sorting, an optional accent color, and a creation timestamp. Columns are indexed by boardId for fast retrieval of all stages belonging to a board.

The cards table holds the individual task records of the application. Each card stores its parent column and board IDs, a title, an optional description, an optional assigned user ID, a fractional order key, an array of associated label IDs, an optional due date timestamp, a completion flag, and an optional priority level (low, medium, high, or urgent). Cards support a full-text search index on the title field, filtered by boardId, to enable fast in-board search.

The labels table stores per-board custom labels. Each label record contains the parent board ID, a name, and a color string. Labels are indexed by boardId and can be attached to any number of cards within the same board.

The activityLogs table records all significant actions taken on a board. Each log entry stores the board ID, an optional card ID, the acting user's ID, an action keyword (such as "created," "moved," "updated," or "deleted"), a human-readable details string, and a creation timestamp. Logs are indexed by both boardId and cardId to support board-level and card-level activity feeds.

The notifications table stores in-app notifications sent to users when tasks are assigned to them. Each notification record links the recipient user, the actor user, the relevant board and card, the notification type (currently "taskAssigned"), the task title, a read flag, and a creation timestamp. Indexes on recipientUserId, isRead, boardId, and cardId support efficient notification retrieval and unread-count queries.

Overall, the FlowBoard schema is designed to support efficient real-time data access, role-based authorization, and a rich task management experience while maintaining clear data relationships and minimal redundancy across all tables.

VII. Features of the System

Sitemap

The FlowBoard sitemap describes the structure and navigation flow of the entire web application. The sitemap is organized into two main sections: the authentication entry point and the authenticated application area. All routes within the authenticated area are protected by an AuthGuard that redirects unauthenticated users to the login page.

The entry point of the application is the Login Page, where users can either sign in with an existing account or register a new one using their email address and password. New users receive a six-digit OTP via email to verify their address before their account is activated. Once authenticated, users are redirected to the Home Page.

The Home Page serves as the user's central dashboard, displaying all boards they own or have been added to as a member. From this page, users can search for a board by name, create a new board through the Create Board Modal, mark boards as favorites, or click into a board to open its full view. Boards are sorted by their most recent update time, and owned versus shared boards are visually distinguished.

The Board Page is the primary workspace of the application. It presents all columns of the selected board in a horizontal Kanban layout, with cards organized within their respective columns. From this page, users can perform the following actions: add new columns; rename, recolor, or delete existing columns; add new cards to any column; drag and drop cards between columns or reorder them within a column; and drag and drop columns to rearrange the workflow stages. Clicking a card opens the Card Detail view, where users can edit the card's title, description, priority, due date, labels, and assignee, and view the card's activity history. The board toolbar includes a search bar for filtering cards by text content and a Filter Panel for filtering by priority, label, and due date. An Activity Feed panel can be toggled to view the board's full action history. The board settings allow the owner to rename the board, change its accent color, manage labels, manage members and invitations, and delete the board.

The Not Found Page is displayed when a user navigates to a route that does not exist within the application, providing a clear error message and a link to return to the Home Page.

  • Not Found Page (404) – Displayed for any unrecognized route; includes a link back to the Home Page.

Through this structure, FlowBoard ensures that all features are logically organized and easily accessible to users, providing a smooth and efficient workflow from authentication through task creation, collaboration, and completion.
